# WORKFLOW_ONE_PAGE

## Purpose

这是最短入口卡。

如果只想花 30 秒找方向，先看这页。

## 1. 先判断任务属于哪类

- 需求梳理：`mode:prd`
- 方案设计：`mode:architecture`
- 新功能开发：`mode:feature`
- 直接实现：`mode:build`
- 技术债治理：`mode:refactor`
- 缺陷修复：`mode:bugfix`
- 问题排查：`mode:debug`
- 文档治理：`mode:docs`
- 测试 / 验证：`mode:test`
- 迁移 / 数据：`mode:migration`
- 联调跟踪：`mode:track`
- 发布核验：`mode:release`
- 审查验收：`mode:review`
