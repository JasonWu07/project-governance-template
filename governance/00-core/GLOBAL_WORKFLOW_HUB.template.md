# GLOBAL_WORKFLOW_HUB

## Purpose

这是全局工作流文档的统一导航页。

如果只先看一份治理文档，就先看这份。

它回答三个问题：

1. 我应该从哪开始？
2. 我这次任务该用哪套规则和模板？
3. 我做完以后该检查什么？

## Short Path

### 如果刚接手一个任务

按这个顺序看：

1. 判断任务属于什么 `mode:*`
2. 读 `WORKFLOW_ONE_PAGE.md`
3. 读 `MODE_TO_SKILLS_PLAYBOOK.md`
4. 找到对应的 `MODE_*_STANDARD_TEMPLATE.md`
5. 按 `WORKFLOW_MINIMUM_VALIDATION_MATRIX.md` 检查最低验证要求
6. 如需交接，套 `MULTI_AGENT_HANDOFF_TEMPLATE.md`

### 如果只想快速知道该调哪些 skill

先看：

- `GLOBAL_SKILLS_QUICK_REFERENCE.md`

### 如果想看全量 skills 台账

看：

- `GLOBAL_SKILLS_INVENTORY.md`

## Recommended Reading Order

### 1. 日常快速开工

1. `WORKFLOW_ONE_PAGE.md`
2. `GLOBAL_SKILLS_QUICK_REFERENCE.md`
3. `MODE_TO_SKILLS_PLAYBOOK.md`
4. 对应 `mode:*` 模板

### 2. 系统理解整套工作流

1. `GLOBAL_SKILLS_INVENTORY.md`
2. `GLOBAL_SKILLS_QUICK_REFERENCE.md`

4. `TASK_TYPE_TO_OUTPUT_INDEX.md`
5. `WORKFLOW_MINIMUM_VALIDATION_MATRIX.md`
6. `WORKFLOW_CLOSURE_CHECKLIST.md`

### 3. 多 agent 协作 / 交接

1. `MULTI_AGENT_HANDOFF_TEMPLATE.md`
2. 对应 `mode:*` 模板

- `mode:review` -> `MODE_REVIEW_STANDARD_TEMPLATE.md`
- `mode:refactor` -> `MODE_REFACTOR_STANDARD_TEMPLATE.md`
- `mode:bugfix` -> `MODE_BUGFIX_STANDARD_TEMPLATE.md`
- `mode:feature` -> `MODE_FEATURE_STANDARD_TEMPLATE.md`
- `mode:docs` -> `MODE_DOCS_STANDARD_TEMPLATE.md`
- `mode:test` -> `MODE_TEST_STANDARD_TEMPLATE.md`
- `mode:migration` -> `MODE_MIGRATION_STANDARD_TEMPLATE.md`
- `mode:track` -> `MODE_TRACK_STANDARD_TEMPLATE.md`
- `mode:release` -> `MODE_RELEASE_STANDARD_TEMPLATE.md`

### 低频但关键模式

- `mode:prd` -> `MODE_PRD_STANDARD_TEMPLATE.md`

- `mode:build` -> `MODE_BUILD_STANDARD_TEMPLATE.md`
- `mode:debug` -> `MODE_DEBUG_STANDARD_TEMPLATE.md`

## End-of-Task Check

任务结束前至少问自己：

1. 我用对 mode 了吗？
2. 我按模板输出了吗？
3. 我达到最低验证要求了吗？
4. 我写清剩余风险了吗？
5. 如果别人接手，他能否继续？
