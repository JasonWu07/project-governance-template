# PULL_REQUEST_STANDARD

## PR Title

建议与 commit title 风格一致：

`<type>: <summary>`

## PR Body Sections

至少包含：

- Summary
- Scope
- Files Changed
- Validation
- Risks
- Rollback Plan

## Writing Rules

- 先写“本次做了什么”
- 再写“本次明确不做什么”
- 验证写清命令或手工步骤
- 风险和回滚不要省略
