# COMMIT_MESSAGE_STANDARD

## One-Line Rule

`<type>: <summary>`

## Recommended Types

- `feat`
- `fix`
- `refactor`
- `docs`
- `test`
- `chore`
- `perf`
- `revert`

## Good Examples

- `feat: add subscription status sync job`
- `fix: repair admin token validation`
- `docs: update QA collaboration guide`

## Writing Rules

- summary 用一句话说明本次改动主结果
- 不写空泛标题，例如 `update`、`change files`
- 一个 commit 尽量只表达一个清晰意图
